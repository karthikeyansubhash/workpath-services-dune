# 07-5. Release Print Failure — Output Stuck in Pending State

> **Analysis Date:** 2026-03-31
> **Related Log:** `07-4.Printfail.txt`
> **Related Code:** `CopyJobAdapter.java`, `CreatingCopyJobState.java`

---

## 1. Symptom

- Store mode: 3-page ADF scan → E2 ScanJob succeeded (jobId: `29b30798`)
- Enumerate → Job displayed normally
- **Release** → E2 CopyJob created successfully (copyJobId: `4a691592`) → **Printing did not start and remained stuck in Pending state**
- Last log entry: `imagesScanned=0, sheetsPrinted=0` — the printer did not output anything

---

## 2. Cause Analysis

### 2.1 Log Evidence

Checking the `CopyOptions` sent to E2 during Release (log line 216):

```json
{
  "originalMediaSource": "miAdf",
  "originalMediaSize": "msAny", 
  "originalPlexMode": "pmSimplex",
  "scanCaptureMode": "scmStandard",
  "scanProgressMode": "spmNone",
  "scaleSelection": "sstFitToPage",
  ...
}
```

**Problem:** Release is a job that only prints already-stored scan data, but `originalMediaSource: miAdf` is included, causing the **E2 firmware to wait for re-scanning originals from the ADF**.

### 2.3 Root Cause

The `getCopyJobTicket()` method **does not distinguish between normal Copy and Release.**

- **Normal Copy:** Scans and prints simultaneously, so `originalMediaSource: miAdf` is needed (to scan from ADF)
- **Release:** Only prints already-scanned stored data, so if `originalMediaSource` is included, the firmware attempts an ADF scan

`getCopyJobTicket()` copies the entire DefaultOptions as the CopyOptions base, so all scan-related options are included and unnecessarily passed to the Release API.

**Log Verification (line 208~212):**
```

```

## 3. Fix Approach

### Approach A: Initialize scan-related fields to null in Release CopyOptions (Recommended)

In `getCopyJobTicketForRelease()`, after calling `getCopyJobTicket()`, clear scan-only (scan-side) fields to null.

**Fix Location:** `CopyJobAdapter.java` — `getCopyJobTicketForRelease()` method

```java
public static CopyJobTicket getCopyJobTicketForRelease(
        CopyAttributes savedCopyAttributes,
        int releaseCopies,
        DefaultOptions defaultOptions) throws IOException {
    SLog.d(TAG, "getCopyJobTicketForRelease : releaseCopies=" + releaseCopies);
    CopyAttributesReader reader = new CopyAttributesReader(savedCopyAttributes);
    CopyJobTicket ticket = getCopyJobTicket(reader, defaultOptions);

    if (ticket.getCopyOptions() != null) {
        // Release only prints stored scan data, so set scan-related options
        // to null to prevent E2 from attempting a re-scan.
        ticket.getCopyOptions().setOriginalMediaSource(null);
        ticket.getCopyOptions().setOriginalMediaSize(null);
        ticket.getCopyOptions().setOriginalPlexMode(null);
        ticket.getCopyOptions().setMediaBindingFormat(null);
        ticket.getCopyOptions().setScanCaptureMode(null);
        ticket.getCopyOptions().setImagePreviewMode(null);
        // scanProgressMode is kept as it may be used for UI display during Release
        SLog.d(TAG, "getCopyJobTicketForRelease : Cleared scan-side options for release");

        // Override copies from release request
        if (releaseCopies > 0) {
            ticket.getCopyOptions().setCopies(
                    CopyTypeMapping.copies.convertWtoE(releaseCopies, defaultOptions.getCopies()));
            SLog.d(TAG, "getCopyJobTicketForRelease : Overrode copies to " + releaseCopies);
        }
    }
    SLog.d(TAG, "getCopyJobTicketForRelease : CopyOptions=" + (ticket.getCopyOptions() != null ? "built" : "null"));
    return ticket;
}
```

### Approach B: Separate Release-specific CopyOptions build method (Alternative)

Instead of calling `getCopyJobTicket()` in `getCopyJobTicketForRelease()`, implement a separate method that selectively sets only print-related fields in CopyOptions.

**Pros:** No possibility of unnecessary scan options being included
**Cons:** Increased code duplication, requires modifications in two places when adding future CopyOptions

> **Recommended: Approach A** — minimal change to existing code, only requires clearing scan-related fields
