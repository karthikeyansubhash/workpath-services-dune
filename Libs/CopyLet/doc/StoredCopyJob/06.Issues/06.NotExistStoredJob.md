# Issue 06: StoredJob Not Appearing After Creation (Flatbed fallback)

## Symptom
- Set `Scan Source` to `ADF` in `CopyAttributes` and submitted a Job (rid `4b48d47e-...`).
- Immediately after, called `enumerateStoredJob` from the app, but `0` jobs were returned from the device.
- When checking in the app, the `ScanJob` status is still stuck in "progress" state.
- The user reported: "I didn't put paper in the ADF, but scanning proceeded from the Flatbed, and the device shows the job as in progress."

## Cause
This is not a Workpath SDK code bug; it is the normal **scan behavior (Fallback & Multi-page Job Build)** of HP FutureSmart device hardware and E2 API.

1. **ADF -> Flatbed Auto Fallback:**
   The E2 Payload clearly included the `"mediaSource":"miAdf"` option. However, when the hardware sensor detects no documents in the ADF, the device automatically falls back to the Flatbed scanner (Glass) to attempt scanning.
2. **Multi-page Wait (Progress Prompt):**
   When scanning via the Flatbed scanner, since multiple physical documents are often merged into one, after scanning the first page, the device displays a prompt dialog on the Control Panel such as **"Do you have another page? / Finish"** and waits indefinitely for user input.
3. **Job Completion Delay:**
   During this wait time, the E2 `ScanJob` status remains in `STARTED` / `ACTIVE` state. In other words, it is in a "waiting for user intervention" state where the scan itself has not fully completed (COMPLETED).
   Since the scan job has not yet completed, the final result `StoredJob` has also not been created on the device's hard drive.
4. **Enumerate Returns 0 Results:**
   Therefore, because the app queried the E2 `/ext/copy/v1/copyAgents/{...}/storedJobs` list during this interim wait period, it naturally received an empty list (`memberIds: []`) as a response since the job has not been stored yet.

## Solution
This is normal E2 and device firmware behavior, so **there is nothing to fix on the SDK side.**
For the scan job to finish and be properly registered in the `StoredJob` list, the physical scanning process must be fully completed.

**Recommended Response (Test Procedure):**
1. When this occurs, look at the actual screen (physical Control Panel) of the printer device — a **"Finish"** button should be displayed. Tapping it will finalize the `ScanJob` status to `COMPLETED`.
2. After finalization, calling `enumerateStoredJob` from the SDK again will successfully show the just-completed scan job in the list.
3. Alternatively, if you **properly load paper into the ADF** before submitting the job, the device will pull in all loaded pages without fallback and automatically process the job to `COMPLETED`, allowing immediate list confirmation via Enumerate without any Control Panel interaction.
