# Issue 04: Release and Delete Fail with "alpha numeric password"

## Symptom
When setting the password to ALPHA-NUMERIC in the Copy sample, the item appears in StoredJob but the password cannot be entered, and when attempting to Release or Delete the job, it cannot proceed further due to password-related errors.

## Cause
This appears to be a bug in CopySample. Looking at the sample app code, there is no password creation functionality for the ALPHA_NUMERIC type.
```java
    @NonNull
    public JobCredentialsAttributes build() throws CapabilitiesExceededException {
        if (this.mStoreJobPasswordType != JobCredentialsAttributes.PasswordType.NONE && TextUtils.isEmpty(this.mStoreJobPassword)) {
            throw new CapabilitiesExceededException("For password type " + this.mStoreJobPasswordType + " a password must be provided");
        } else if (this.mStoreJobPasswordType == JobCredentialsAttributes.PasswordType.NUMERIC && !TextUtils.isDigitsOnly(this.mStoreJobPassword)) {
            throw new CapabilitiesExceededException("For password type " + this.mStoreJobPasswordType + " only digits are allowed in a password");
        } else {
            return new JobCredentialsAttributes(this);
        }
    }

    public static enum PasswordType {
        NONE,
        NUMERIC,
        ALPHA_NUMERIC;
    }
```

## Solution
Sample app modification required.
