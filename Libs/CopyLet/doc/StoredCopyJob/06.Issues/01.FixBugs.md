# Bug Fix Record

## Appendix A. Release 400 Error Detailed Analysis

### Error Log

```
2026-03-26 21:10:27.721  [SAMPLE]Copy  D  Release StoredJob submitted with rid = 7f0961c7-cb0f-467c-96c6-a5274a8a62ae
2026-03-26 21:10:28.370  [SDK]Joblet   D  JobService state: fail
2026-03-26 21:10:28.371  [SAMPLE]Copy  I  onFail: Received rid=7f0961c7-cb0f-467c-96c6-a5274a8a62ae,
  [Code:RESULT_FAIL,ErrorCode:SERVICE_ERROR,
   Cause:HTTP request failed. 400: {
     "errors":[{
       "code":"badParametersError",
       "message":"Invalid request body for releasing stored job: CopyOptions are missing Password is empty "
     }],
     "httpStatusCode":400
   }]
```

### Root Cause

E2 Release endpoint (`POST .../storedJobs/{jobId}/release`) returned HTTP 400. Two issues:

**Issue 1: CopyOptions missing**

In `processReleaseJobIntent()`, when `savedCopyAttributes` is null (for existing stored jobs created before the code was deployed, or when the jobId does not exist in PreferenceStorage), `releaseRequest.setCopyOptions()` is not called, causing CopyOptions to be sent as null.

```java
// Before fix (CopyOptions not set when savedCopyAttributes is null):
if (savedCopyAttributes != null) {
    DefaultOptions defaultOptions = copyJobService.getDefaultOptions(packageName);
    if (defaultOptions != null) {
        CopyJobTicket ticket = CopyJobAdapter.getCopyJobTicketForRelease(...);
        releaseRequest.setCopyOptions(ticket.getCopyOptions());
    }
}
// → savedCopyAttributes == null → CopyOptions = null → E2 400
```

**Issue 2: Password empty**

The existing code does not call `setJobPassword()` when password is null. E2 expects a password field even for stored jobs without a password.

```java
// Before fix:
if (credentials != null && credentials.getStoreJobPassword() != null) {
    releaseRequest.setJobPassword(credentials.getStoreJobPassword());
}
// → password null → setJobPassword not called → E2 "Password is empty"
```

### Changes

```java
// After fix — CopyOptions fallback:
DefaultOptions defaultOptions = copyJobService.getDefaultOptions(packageName);
if (savedCopyAttributes != null && defaultOptions != null) {
    CopyJobTicket ticket = CopyJobAdapter.getCopyJobTicketForRelease(...);
    releaseRequest.setCopyOptions(ticket.getCopyOptions());
} else if (defaultOptions != null) {
    // Fallback: DefaultOptions → CopyOptions JSON conversion
    releaseRequest.setCopyOptions(CopyJobAdapter.convertDefaultOptionsToCopyOptions(defaultOptions));
}

// After fix — conditional transmission based on passwordType:
if (credentials != null
        && credentials.getStoreJobPasswordType() != null
        && credentials.getStoreJobPasswordType() != PasswordType.NONE) {
    releaseRequest.setJobPassword(password != null ? password : "");
}
// + Applied @JsonInclude(NON_NULL) to ReleaseStoredJobRequest → null fields excluded from JSON
```

`CopyJobAdapter.convertDefaultOptionsToCopyOptions()` extracts the existing DefaultOptions→CopyOptions JSON conversion logic from inside `getCopyJobTicket()` as a public method.

> **Note:** Appendix A is the analysis record from the initial discovery. The subsequently discovered passwordType=NONE related issue and final fix details are recorded in **Section 8 Bug Fix #3**.

**Discovery Date:** 2026-03-26
**Symptom:** E2 returns HTTP 400 on Release request

// CopyOptions fallback:
if (savedCopyAttributes != null && defaultOptions != null) {

### Bug Fix #2 — Delete 400: Request body expected, but not found

**Discovery Date:** 2026-03-26
**Symptom:** E2 returns HTTP 400 on Delete request
```
badParametersError: Request body expected, but not found.
```

**Root Cause:**

`RemoveStoredJobOperationResourceFacade.executeAsync()` calls `ResourceFacadeHelper.executeResourceOperationAsync()` with `resource=null`. `createResourceAsync()` sends a POST without body when `resource=null`, but the E2 remove endpoint requires a POST that includes at least an empty body (`{}`).

```java
// Before fix — POST without body:
return ResourceFacadeHelper.executeResourceOperationAsync(
    getHttpClient(), createRequestUri(...), accessToken, null, StoredJob_Remove.class);
```

**Changes:**

| File | Change |
|------|--------|
| `RemoveStoredJobOperationResourceFacade.java` | Pass an empty `new StoredJob_Remove()` object as body instead of `null` |

```java
// After fix — sending empty JSON body({}):
StoredJob_Remove emptyBody = new StoredJob_Remove();
return ResourceFacadeHelper.executeResourceOperationAsync(
    getHttpClient(), createRequestUri(...), accessToken, emptyBody, StoredJob_Remove.class);
```

**Rationale:** When `resource != null`, `createResourceAsync()` generates a JSON body using `ObjectMapper.writeValueAsString(resource)` and includes it in the POST. An empty `StoredJob_Remove` object serializes to `{}`, satisfying E2's body requirement.

**Verification:** Manual test passed (Delete confirmed successful)

### Bug Fix #3 — Release 400: Password is empty (passwordType=NONE job)

**Discovery Date:** 2026-03-26
**Symptom:** E2 returns HTTP 400 when Releasing a stored job with passwordType=NONE
```
badParametersError: Invalid request body for releasing stored job: Password is empty
```

**Root Cause:**

Bug Fix #1 modified the code to always send password (`null→empty string`), but E2 returns a "Password is empty" error **when the `jobPassword` field itself exists** (including empty string) **for passwordType=NONE jobs**.

In other words, E2's validation logic:
- passwordType=NUMERIC/ALPHA_NUMERIC → `jobPassword` field **required** (error if empty)
- passwordType=NONE → `jobPassword` field **forbidden** (error if present)

Additionally, Jackson `ObjectMapper` default settings serialize null fields as `"jobPassword": null`, so even if `setJobPassword()` is not called, `jobPassword: null` may be included in the JSON.

**Changes (2 files):**

| File | Change |
|------|--------|
| `CreatingCopyJobState.java` | Only call `setJobPassword()` when passwordType is not NONE. If NONE or credentials are absent, do not call (keep null) |
| `ReleaseStoredJobRequest.java` | Added `@JsonInclude(JsonInclude.Include.NON_NULL)` annotation — completely excludes null fields from JSON serialization |

```java
// CreatingCopyJobState.java — after fix:
if (credentials != null
        && credentials.getStoreJobPasswordType() != null
        && credentials.getStoreJobPasswordType() != JobCredentialsAttributes.PasswordType.NONE) {
    String password = credentials.getStoreJobPassword();
    releaseRequest.setJobPassword(password != null ? password : "");
} else {
    // passwordType=NONE: jobPassword not set → excluded from JSON
}
```

```java
// ReleaseStoredJobRequest.java — @JsonInclude added:
@JsonInclude(JsonInclude.Include.NON_NULL)
public class ReleaseStoredJobRequest extends E2Type {
    // If jobPassword is null, it will not be included in JSON
}
```

**Result JSON Comparison:**

| Scenario | Before Fix | After Fix |
|----------|-----------|-----------|
| passwordType=NUMERIC, password="1234" | `{"copyOptions":{...}, "jobPassword":"1234"}` | `{"copyOptions":{...}, "jobPassword":"1234"}` |
| passwordType=NUMERIC, password=null | `{"copyOptions":{...}}` (jobPassword missing) | `{"copyOptions":{...}, "jobPassword":""}` |
| passwordType=NONE | `{"copyOptions":{...}, "jobPassword":""}` → **400 error** | `{"copyOptions":{...}}` (jobPassword excluded) → **success** |

**Verification:** Manual test passed — confirmed Release success for both passwordType=NONE and passwordType=NUMERIC jobs

### Bug Fix #4 — Delete 403: Incorrect password for stored job

**Discovery Date:** 2026-03-30
**Symptom:** E2 returns HTTP 403 error when requesting deletion of a Stored Copy Job with a password set
```
noAuthorityError: Incorrect password for stored job
```

**Root Cause:**
`RemoveStoredJobOperationResourceFacade` was hardcoded to send an empty object (`{}`) without a password field as the Body for delete requests, and on the `CreatingCopyJobState` side, the interface (signature) to pass the entered password to the E2 service was not implemented.

**Changes:**

| File | Change |
|------|--------|
| `RemoveStoredJobRequest.java` | Added `@JsonInclude(JsonInclude.Include.NON_NULL)` |
| `RemoveStoredJobOperationResourceFacade.java` | Added overload to accept `RemoveStoredJobRequest` and send it as body |
| `IDeviceCopyJobService.java` & `Standard` implementation | Added `deleteStoredJob()` overload accepting `RemoveStoredJobRequest` | 
| `CreatingCopyJobState.java` | In `processDeleteJobIntent()`, parses credentials from `DeleteRequestIntent` and passes Password to E2 API call |

**Verification:** Unit tests passed and manual test passed (confirmed normal Delete for password-protected job)
