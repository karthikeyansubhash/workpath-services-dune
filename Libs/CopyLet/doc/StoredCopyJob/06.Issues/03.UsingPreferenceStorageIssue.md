# Issue: Enumerated Stored Jobs Missing Scan Attributes

## Overview
When `enumerateStoredJob` is called, the UI lists the stored copy jobs with default copy attributes (e.g., `ColorMode.DEFAULT`, `Duplex.NONE`, `ScanSize.DEFAULT`), despite the fact that these attributes were correctly captured and stored during the Create phase.

## Root Cause Analysis
The E2 `StoredJob` interface for scanning/JobStorage only stores and returns a limited set of metadata (e.g., `folderName`, `jobId`, `jobName`, `totalPages`). It does not persist properties that belong to the original `CopyAttributes` scan configuration.
In `CopyJobAdapter.java`, the `convertToStoredJobInfoList` method was creating standard `StoredJobInfo` objects completely reliant on the raw details retrieved from the E2 service without taking into consideration the local `PreferenceStorage` used in the SDK. As a result, stored values like user-defined colour modes and duplex configurations were dropping back to defaults.

## Fix / Resolution
1. **Pass Context to Adapter**
   In `OXPCopyletContentProvider.java`, updated the `ENUMERATE_JOBS` handling to pass the `Context` (`getContext()`) to `CopyJobAdapter.convertToStoredJobInfoList()`.
2. **Merge from PreferenceStorage**
   In `CopyJobAdapter.java`, we now iterate over the collection members and retrieve the corresponding `CopyAttributes` from `StoredCopyJobPreferenceStorage` using the item `jobId`.
3. **Use CopyAttributesReader**
   We wrap the retrieved properties in a `CopyAttributesReader` to correctly extract the values (`getColorMode`, `getScanSize`, `getPrintDuplex`, `getCopies`), overriding the E2 defaults inside `convertStoredJob()`.

## Verification
After the changes, enumerate loops merge the raw E2 summary and our locally stored preferences:
```
[WS]JobISS/Cre/Copy      D  processStoredCopyJobIntent : CopyAttributes saved to PreferenceStorage, key=4c52310d...
[WS]Copylet/JAdap        D  convertStoredJob WP : applying saved attributes from PreferenceStorage
[WS]Copylet/JAdap        D  convertStoredJob WP : jobId=4c52310d..., [colorMode=GRAY, scanSize=ISO_A4, duplex=BOOK, copies=1, totalPages=12]
```
The SDK behaves precisely as originally designed, restoring full state information on enumeration despite limitations with the underlying E2 protocol.
