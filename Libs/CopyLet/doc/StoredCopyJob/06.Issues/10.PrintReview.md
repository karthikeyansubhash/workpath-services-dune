"originalMediaSize": null,
"originalMediaSource": null,
"originalPlexMode": null,
"scanCaptureMode": null

→ scan-side fields sent as `null`

### 4.3 E2 Response CopyOptions

In the E2 response:
```json
"originalMediaSource": "miAdf",
"originalMediaSize": "msAny",
"originalPlexMode": "pmSimplex"
```
→ E2 fills null values with its own defaults. The null values sent by SDK and E2 response values are independent.

## 6. Known Issues and Follow-up Tasks

### 6.1 Actual Hardware Device Testing Required
Although it succeeded on the simulator (latest firmware `6.39.0.283`), the actual device (older firmware `6.39.0.156`) showed the `processing/waiting` stuck phenomenon. Retesting after updating the actual device to the latest firmware is needed.

### 6.2 Effect of scan-side null Clearing Unclear
Whether the clearing of `setOriginalMediaSource(null)` etc. actually affects firmware behavior requires A/B comparison testing (checking success even without null clearing applied).

### 6.3 MonitoringStoredCopyJobState Cleanup Incomplete
The issue where `StoredCopyJobPreferenceStorage.remove()` is not called upon stored job failure/cancellation (Issue 1 from previous analysis) requires a separate fix.
